# 数据与图表质量规则

仅在文稿包含公式、派生数字、模拟结果或数据图表时读取本文件。

## 数据证据最小契约

每个结果型数字至少明确：

- 数值与单位。
- 基线或比较对象。
- 时间窗口或仿真场景。
- 数据来源、计算方法，或「示意参数」「模拟结果」标识。

派生数字必须能追溯到输入参数、公式和单位。跨页复用参数时保持名称、数值和单位一致；面积、流量、时间、能耗、百分比与百分点要分别检查量纲。

公式混合不同量纲时，说明归一化方法和权重定义。无法验证的精确结果不要写成已落地成效。

## 图表准入

| 图表 | 必须满足 | 不满足时改用 |
| --- | --- | --- |
| 折线图 | 时间、连续顺序或确有趋势的有序阶段 | 条形图、KPI、并列清单 |
| 环形图 | 同一整体、同一单位，分项合计等于整体或 100% | KPI 卡、条形图 |
| 堆叠条形图 | 每组具有相同构成口径 | 分组条形图、矩阵 |
| 瀑布图 | 同一单位、可加减，首尾总量可由中间项解释 | 前后对比、小倍图 |
| 四象限图 | 两个明确尺度，坐标归一到 0–100，气泡尺寸有统一含义 | 决策矩阵、排序表 |
| 漏斗图 | 同一指标逐层筛选，数值单调不增 | 流程图、时间线 |

禁止把「图表数量」作为目标。先验证数据关系，再选择图表。

## Theme13 数值约定

- `theme13_page025`：最后一个点不得是前面类别的合计；总量放在标题或主指标中。
- `theme13_page026`：`centerValue` 必须表示可核对的总体，如 `100%` 或分项合计。
- `theme13_page028`：填写非空 `unit`；首尾项作为总量，中间损耗写负数、增益写正数，最终值等于基础值加各增减项。
- `theme13_page029`：`scoreX`、`scoreY` 使用 0–100；`size` 建议 18–36，不得直接写 1–5 评分。
- `theme13_page030`：`stages[].value` 必须单调不增。时间节点使用流程或 Roadmap，不使用漏斗。

## 渲染后检查

- 标题没有单字或双字孤行。
- 图表占据主要视觉区域，不缩在角落。
- 坐标、标签、气泡和图例不重叠或聚集。
- 图表标题与数据结论一致。
- `insight` 是由当前数据得出的判断，不是模板默认句。
