# 单页内容构图规则

主题负责统一视觉语言；构图负责把原始材料转成高层可以快速判断的页面。一页只承担一个主要认知任务，确保 5 秒能读出结论、30 秒能看懂依据。

## 选择顺序

1. 定义页面任务：理解、比较、诊断、解释、建模、排序或行动。
2. 先写结论式标题，不用只有主题词的标题。
3. 将段落转成卡片、指标、矩阵、模型、时间线或行动项。
4. 选择与任务匹配的版式，并设置唯一视觉主角。
5. 为数据、模型和比较页补充一条 `insight`，说明「这意味着什么」。

## 页面任务映射

| 页面任务 | 推荐结构 | 查询角色 |
| --- | --- | --- |
| 全局判断 | 执行摘要、发现卡片 | `overview` |
| 证据 | Hero 指标、趋势、数据页 | `evidence` |
| 取舍 | 双栏对比、决策矩阵 | `comparison` / `decision` |
| 问题根因 | 三层诊断模型 | `diagnosis` |
| 机制说明 | 流程与步骤 | `explanation` |
| 要素关系 | 分层架构、协作模型 | `model` |
| 阶段推进 | Roadmap、里程碑 | `timeline` |
| 下一步 | 优先级行动卡片 | `actions` |

## 生成前的构图计划

在输出目录创建 `composition-plan.json`，但不要写入 `goal.json`：

```json
{"pages":[{"page":3,"pageJob":"decision","messageTitle":"先完成三项基础决策，再扩大建设范围","visualTranslation":"priority action cards","primaryFocus":"三项决策","insightStrip":"先验证规则，再扩大投入","imagePolicy":"CSS/SVG only"}]}
```

## 检查清单

- 标题是否直接表达判断。
- 是否只有一个主要论点和一个视觉主角。
- 内容是否已从段落翻译成合适的视觉结构。
- 数据、模型、比较页是否解释了结论。
- 是否存在堆砌文本、无关默认文案或空媒体槽。
